# PDF Deep Reader Skill

## 触发条件

当用户提到以下关键词时启动：
- 「读这个PDF」「分析这份文档」「解读这份研报」
- 「看看这个报告」「提取要点」「总结一下」
- 直接提供 PDF 文件路径

---

## 执行流程

```
接收 PDF 文件
    ↓
使用 Read 工具读取 PDF 内容
    ↓
结构化解析（识别章节、表格、图表说明）
    ↓
根据文档类型选择分析模板
    ↓
输出深度分析报告
```

---

## 文档类型识别与分析策略

### 1. 投资研报类

**识别特征：** 包含「买入/卖出/持有」「目标价」「EPS预测」「行业评级」等关键词

**分析框架：**
```
1. 核心结论提取
   - 评级变化（上调/下调/维持）
   - 目标价及测算逻辑
   - 关键假设条件

2. 逻辑链梳理
   - 宏观前提 → 行业判断 → 公司定位 → 估值结论
   - 标注每个环节的数据支撑

3. 风险因素
   - 报告明确提示的风险
   - 报告隐含但未强调的风险

4. 可验证节点
   - 提取时间节点（如：Q1财报、政策出台）
   - 提取数据验证点（如：产能达产、订单落地）

5. 与现有认知对比
   - 与 CLAUDE_MEMORY.md 中的观点交叉验证
   - 标注一致/冲突之处
```

### 2. 政策文件类

**识别特征：** 包含「通知」「意见」「办法」「规划」等关键词，或来源为政府机构

**分析框架：**
```
1. 政策要点速览
   - 发文机构及层级
   - 适用范围
   - 生效时间

2. 关键条款拆解
   - 定量目标（数字、比例、时限）
   - 约束条件（禁止项、限制项）
   - 激励措施（补贴、税收优惠）

3. 利好/利空判断
   - 直接受益行业/公司
   - 间接影响链条
   - 潜在风险点

4. 历史政策对比
   - 与前次政策的变化
   - 执行力度预判

5. 投资含义
   - 短期交易机会
   - 中长期配置逻辑
```

### 3. 学术/技术文档类

**识别特征：** 包含「Abstract」「Methods」「Results」或 RFC、技术白皮书格式

**分析框架：**
```
1. 一句话总结
   - 这篇文档解决什么问题？

2. 核心创新点
   - 与现有方案的差异
   - 技术路线选择

3. 关键数据/结论
   - 实验结果
   - 性能指标

4. 局限性与适用场景
   - 作者承认的限制
   - 潜在的应用边界

5. 实践启发
   - 对当前工作的借鉴意义
```

### 4. 财务报告类

**识别特征：** 包含「资产负债表」「利润表」「现金流量表」或年报/季报格式

**分析框架：**
```
1. 核心财务指标
   - 营收/利润及增速
   - 毛利率/净利率变化
   - ROE/ROA 趋势

2. 现金流质量
   - 经营现金流 vs 净利润
   - 自由现金流状况
   - 资本开支方向

3. 资产负债健康度
   - 负债率及结构
   - 应收账款周转
   - 存货变化

4. 业绩驱动因素
   - 量价拆分
   - 成本变化
   - 一次性损益

5. 管理层指引
   - 未来展望
   - 战略方向
   - 潜在风险提示
```

## 特殊处理规则

### 表格数据
- 完整保留原始表格
- 计算同比/环比变化
- 标注异常值

### 图表说明
- 提取图表标题和数据趋势
- 补充文字描述

### 多文档对比
如果用户要求对比多个文档：
1. 建立统一对比维度
2. 列表对比关键差异
3. 给出综合判断

---

## 质量检查

每次输出前确认：
- [ ] 核心结论是否明确
- [ ] 数据引用是否准确
- [ ] 逻辑链条是否完整
- [ ] 是否遗漏重要信息
- [ ] 行动建议是否可操作
