# 投资分析框架 Skill

## 触发条件

当用户提到以下关键词时启动：
- 「分析一下 XXX」「怎么看 XXX」「XXX 逻辑是什么」
- 「产业分析」「板块分析」「行业研究」
- 「资金流向」「主力动向」「趋势判断」
- 「验证一下」「反驳一下」「有没有问题」

---

## 核心方法论：资金流向 → 因果链 → 质疑反例

### 三步分析法

```
Step 1: 资金流向验证
    ↓ 用数据说话，不看观点
Step 2: 因果链条构建
    ↓ 政策→产业→公司→估值
Step 3: 质疑与反例
    ↓ 主动找反面证据
```

---

## 产业分析四步框架

### 第一步：政策拆解

```markdown
## 政策层面分析

### 1. 政策梳理
- 近期相关政策（按时间倒序）
- 政策发文层级（国务院/部委/地方）
- 政策力度评估（强制/引导/鼓励）

### 2. 定量目标提取
- 产能/产量目标
- 市场规模预期
- 补贴/税收政策

### 3. 政策边际变化
- 与前次政策对比
- 预期差分析
- 后续政策推演
```

### 第二步：资金验证

```markdown
## 资金流向验证

### 1. 机构动向
- 北向资金近期流向
- 公募基金持仓变化
- 社保/险资配置动态

### 2. 产业资本信号
- 定增/回购/减持
- 大股东增持
- 并购重组

### 3. 一二级市场联动
- 一级市场投融资热度
- IPO/再融资节奏
- 产业链上下游资本动作

### 数据核验清单
- [ ] 主力净流入是否持续
- [ ] 机构持仓是否提升
- [ ] 产业资本是否进场
```

### 第三步：趋势判断

```markdown
## 趋势判断

### 1. 产业周期定位
- 当前周期阶段（导入/成长/成熟/衰退）
- 周期驱动因素
- 周期持续时间预判

### 2. 景气度跟踪
- 量：产销数据、订单情况
- 价：价格走势、库存水平
- 利：毛利率、净利率变化

### 3. 竞争格局演变
- 行业集中度变化
- 龙头份额提升/下降
- 新进入者威胁

### 趋势强度评分（1-10）
- 政策支持度：___
- 资金认可度：___
- 业绩确定性：___
- 估值安全性：___
- 综合得分：___
```

### 第四步：预测+验证节点

```markdown
## 预测与验证

### 1. 核心预测
- 预测内容：[具体描述]

- 预测依据：[逻辑支撑]

### 2. 验证节点设定

- **乐观情景**：[条件] → [结果] → [操作建议]
- **中性情景**：[条件] → [结果] → [操作建议]
- **悲观情景**：[条件] → [结果] → [操作建议]

### 4. 止盈止损规则
```

## 质疑与反例检验

### 强制执行的反向思考

每次分析完成后，**必须**回答以下问题：

```markdown
## 魔鬼代言人检验

### 1. 逻辑漏洞检查
- 我的核心假设是什么？
- 如果假设错误，会发生什么？
- 有没有混淆相关性和因果性？

### 2. 反面证据搜集
- 有哪些数据/观点与我的结论相反？
- 看空方的主要论据是什么？
- 历史上类似情况的反例？

### 3. 预期差审视
- 市场是否已经Price In？
- 我的信息是否有优势？
- 还有什么是我不知道的？

### 4. 最坏情况推演
- 如果判断完全错误，最大损失是多少？
- 有没有黑天鹅风险？
- 安全边际是否足够？
```

## 与记忆系统联动

### 读取参考
每次分析时自动参考：
- `CLAUDE_MEMORY.md` - 当前持仓与观点
- `核心选股逻辑精炼版.md` - 历史分析框架
- `stock_cache/` - 已缓存的股票数据

### 写入更新
分析完成后提示用户是否更新：
- 新的板块观点 → 更新 `CLAUDE_MEMORY.md`
- 新的验证节点 → 添加到跟踪列表
- 与历史观点冲突 → 标注差异
