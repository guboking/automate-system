# 思维工具

> 一生二，二生三，三生万物。

---

## 本质

```
角度 → 问题 → 答案
```

角度选对了，问题变简单，答案自然浮现。
角度选错了，越努力越迷失。

---

## 一、找角度

### 升维

任何问题，先往上看一层：

```
你看到的问题
    ↑
这个问题存在的背景/系统
    ↑
这个系统存在的更大背景
```

**例**：
- 股价涨跌 ← 供需 ← 政策 ← 国家战略 ← 世界格局
- Bug在哪 ← 代码逻辑 ← 架构设计 ← 需求理解 ← 用户真正想要什么

### 换维

同一个问题，换个维度表达：

```
代数 → 几何
时间 → 空间
局部 → 全局
```

**例**：
- 一元三次方程难解 → 画成曲线，交点自己出来
- 找不到路径 → 画成图，一眼看出

---

## 二、找根本

### 问"为什么"

```
表面问题：怎么从 A 到 B？
    ↓ 为什么要去B？
根本问题：传递信息
    ↓ 传递信息一定要"去"吗？
更优解：打电话
```

### 问"谁决定"

```
表面：这个结果会怎样？
    ↓
根本：谁在决定这个结果？他想要什么？
```

### 问"什么不变"

```
变化太多，抓不住
    ↓
找到不变的锚点，其他都是波动
```

## 三、层级思维

任何系统都有层级：

```
规则层    ← 决定游戏怎么玩
策略层    ← 决定资源怎么配
执行层    ← 决定事情怎么做
细节层    ← 决定动作怎么优化
```

**原则**：
- 上层决定下层的边界
- 在错误的层级努力，事倍功半
- 先定位问题在哪一层，再在那一层解决

## 四、目的-路径

```
┌─────────────────────────────────────────┐
│                                         │
│   目的决定路径，不是路径决定目的。      │
│                                         │
│   先确认目的地对不对，                  │
│   再决定怎么走。                        │
│   甚至确认"走"这个动作是否必要。        │
│                                         │
└─────────────────────────────────────────┘
```

## 五、应用

这个工具可以用于：

| 领域 | 应用 |
|------|------|
| 投资 | 先看格局政策，再看供需技术 |
| 技术 | 先理解需求本质，再写代码 |
| 商业 | 先看用户要什么，再做产品 |
| 人生 | 先问想要什么生活，再选路径 |
| 学习 | 先建框架，再填细节 |
| 沟通 | 先理解对方要什么，再组织语言 |
| 解决问题 | 先问问题的根本是什么 |

## 六、心法

```
先定性，再定量。
先选路，再走路。
先升维看全貌，再降维做细节。

找到根本，答案自然浮现。
纠缠表象，越分析越乱。

最高级的解题：让问题消失。
```

---

## 七、一句话

> **不是"怎么解这道题"，是"这道题的本质是什么"。**
