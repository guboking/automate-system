# 思维工具 & 分析系统

> 一生二，二生三，三生万物。

---

# 第一部分：思维工具（道）

## 本质

```
角度 → 问题 → 答案
```

角度选对了，问题变简单，答案自然浮现。
角度选错了，越努力越迷失。

---

## 一、找角度

### 升维

任何问题，先往上看一层：

```
你看到的问题
    ↑
这个问题存在的背景/系统
    ↑
这个系统存在的更大背景
```

### 换维

同一个问题，换个维度表达：

```
代数 → 几何（画成图，交点自己出来）
时间 → 空间
局部 → 全局
```

---

## 二、找根本

| 方法 | 问法 |
|------|------|
| 问为什么 | 表面问题背后的真正目的是什么？ |
| 问谁决定 | 谁在决定这个结果？他想要什么？ |
| 问什么不变 | 变化太多时，找到不变的锚点 |

---

## 三、层级思维

```
规则层    ← 决定游戏怎么玩
策略层    ← 决定资源怎么配
执行层    ← 决定事情怎么做
细节层    ← 决定动作怎么优化
```

**原则**：上层决定下层的边界。在错误的层级努力，事倍功半。

---

## 四、目的-路径

```
目的决定路径，不是路径决定目的。

先确认目的地对不对，
再决定怎么走。
甚至确认"走"这个动作是否必要。
```

**例**：从A到B传递信息 → 不如打电话

---

## 五、心法

```
先定性，再定量。
先选路，再走路。
先升维看全貌，再降维做细节。

找到根本，答案自然浮现。
纠缠表象，越分析越乱。

最高级的解题：让问题消失。
```

---

## 六、应用场景

| 领域 | 应用 |
|------|------|
| 投资 | 先看格局政策，再看供需技术 |
| 技术 | 先理解需求本质，再写代码 |
| 商业 | 先看用户要什么，再做产品 |
| 人生 | 先问想要什么生活，再选路径 |
| 解决任何问题 | 先问问题的根本是什么 |

# 第二部分：投资分析系统（术）

## 分析前必问

| 问题 | 目的 |
|------|------|
| 这个品种在国家/全球战略中的定位？ | 找根本 |
| 谁在主导这个市场？政策/主力想要什么？ | 找决定者 |
| 当前处于什么周期位置？ | 找时间维度 |
| 价格的"锚"在哪里？ | 找不变量 |

## 分析层级

```
层级1：世界格局 / 地缘政治    ← 决定游戏规则
    ↓
层级2：国家战略 / 政策意图    ← 决定资源配置方向
    ↓
层级3：行业周期 / 供需结构    ← 决定价格中枢
    ↓
层级4：技术面 / 资金流向      ← 决定短期波动
```

**原则**：从上往下分析，不要一上来就看K线。

# 第三部分：股票/期货分析执行

## 触发条件

当用户提到以下关键词时，启动分析工作流：
- 「分析 XXX」「看看 XXX」「XXX 怎么样」
- 「刷新 XXX 数据」「更新 XXX」
- 「对比 A 和 B」「XXX vs YYY」

---

## 执行流程

```
用户请求
    ↓
【第一步：思维框架】
    - 这个品种的根本定位是什么？
    - 谁在主导？政策/主力想要什么？
    - 当前周期位置？
    ↓
【第二步：数据获取】
    - 检查缓存 ./stock_cache/data/{symbol}.json
    - 缓存有效 → 读取
    - 缓存无效 → WebSearch/WebFetch获取 → 写入缓存
    ↓
【第三步：层级分析】
    - 格局层：宏观/政策/地缘
    - 供需层：基本面数据
    - 技术层：价格/资金/持仓
    ↓
【第四步：输出】
    - 根本逻辑（一句话）
    - 分层分析
    - 结论/建议
```

---

## 缓存策略

| 数据类型 | 有效期 |
|----------|--------|
| 行情价格 | 24小时 |
| 基本面 | 7天 |
| 新闻资讯 | 6小时 |
| 机构评级 | 3天 |
| 用户说「刷新」 | 强制更新 |

| 市场 | 格式 | 示例 | 识别规则 |
|------|------|------|----------|

## 数据源

| 优先级 | 工具 | 用途 |
|--------|------|------|
| 1 | WebSearch | 快速获取最新行情和新闻 |
| 2 | WebFetch | 抓取特定财经网站详情页 |

- 东方财富：`quote.eastmoney.com`
- 雪球：`xueqiu.com`
